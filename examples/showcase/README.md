The new `atlas_canonical_ocio_{oceancastle,spacehangar,ghosttown}_dcc` trio turns the three ACEScg thumbnails into compact production graphs: explicit OCIO input conversion, localized SDXL or LaMa removal, mask and cleanplate approval previews, a **second depth solve on the cleanplate for continuous hidden support geometry**, the SAM3-matted original foreground at its photographed depth, ACES Output Desk metadata, and **content-keyed per-layer quadric retopology shared by NukeLayers and MayaLayers**. Re-solving the cleanplate is deliberate: extending a far depth band beneath the removed car/castle makes the object float over a vertical cliff during camera moves. They are generated by `tools/generate_canonical_ocio_dcc_workflows.py` and use separate Nuke/Maya output folders to avoid concurrent OBJ writes. Large removals remain a creative approval point: if the preview changes perspective, repeats the occluder, or smears texture, substitute an artist-painted cleanplate before export.

For screenshots and demos, `marketing/workflows/atlas_marketing_ocio_*_dcc_workflow.json` uses an approved 4K cleanplate in the same OCIO/DCC graph and exposes it through a large **HERO OUTPUT** preview. ## What the full-coverage sweep found (notes on what isn't perfect)

1. **`AtlasInput.use_vlm=True` did not force 4 band layers when `layers=0`**
   â€” the force only fires for `layers>0`; you silently get the single-relief
   path. The workflow now sets `layers=4, sky=True` explicitly. (Doc/UX trap;
   candidate for a follow-up: force regardless.)
1b. **VLM per-band SAM scoping can gut a band** â€” a PARTIAL segment match
   (mid band: 4.6% coverage on this plate) legitimately keeps the scope,
   leaving the rest of the band's range with NO geometry, which exposes the
   behind-band's `fill_occluded` smear. New `AtlasInput.vlm_scope` widget
   (APPENDED â€” the mid-list first attempt re-proved the positional
   widgets_values rule live): OFF keeps the VLM's bands/geometry but leaves
   layers band-only. The camera-move workflow ships it OFF; band_mid went
   from 6k verts / 4.6% matte to 96k verts / 66% matte.
2. **A VLM band plan could emit a ZERO-WIDTH far band** (`far=[1.0,1.0]`,
   observed live from gemma on the ghost-town plate): the old monotonic
   check accepted `b3=1.0`, the far card collapsed, the bg band ran to
   infinity and its `fill_occluded` diffusion smear outranked every real
   layer (farthest-highest priorities). **Fixed in
   `assessor.staged_layer_bands`** â€” boundaries must now be strictly inside
   (0,1) and strictly ordered, else all reset to the fixed slots. Pinned by
   `tests/test_assess_image.py`.
3. **`AtlasPredictHiddenGeometry` without `restrict_mask` substitutes far too
   much** (50.6% of the wreck frame, including visible background â€” the
   node's own report warns). The workflow wires a SAM3 segment of the
   occluder; 23.8% after.
4. **`AtlasMergeGeometry` doesn't dedupe `projection_ground`** â€” a 3-way
   merge carries 3 coincident ground planes. Harmless under ًں“½ (identical
   texels by ray); a cosmetic dedupe would still be nice.
5. **`SaveImageExtended` writes ONE file per queue for an IMAGE batch** in
   this configuration â€” not a per-frame sequence. The authoritative JPEG
   sequence is extracted from the baked `client_data` (the frames are already
   JPEG bytes); the saver is kept as an in-graph preview of frame delivery.
6. **Stale-node verdicts:** `AtlasLoadImageSolveCamera` â€” now formally
   deprecated (`DEPRECATED = True` + "(Deprecated)" display name; file-path
   input predates the tensor solve nodes; can't join image chains; kept
   registered so saved workflows load, removal in a later release; still
   shown in the solve lab's "legacy corner" with a note).
   `AtlasDepthAnything` â€” keep, but it is a *preview* (normalized IMAGE,
   metric data discarded); never wire it toward geometry. Every other node
   earned its place.
7. **`AtlasVLMScaleCues` returned zero references on the alley plate**
   (gemma emitted no door/person cues) â€” the confirm-gated
   `AtlasApplyScaleReferences` correctly adopted nothing. The reference-object
   node (`AtlasReferenceScaleSolve`) remains the dependable tier-1 path.
