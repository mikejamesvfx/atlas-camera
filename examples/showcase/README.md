# Atlas v1 showcase workflows

Nine plates, nine stories — and together they exercise **every Atlas node**
(all 60 classes; the 2026-07-16 coverage audit found 33 of them demonstrated
by no workflow, which is what this set fixes). Generated by
`tools/generate_showcase_workflows.py` (widget lists derived from the live
`/object_info` — edit the generator, never the JSON), validated by
`tools/validate_ui_workflow.py`, runnable headless via
`tools/run_ui_workflow.py`. Like `experimental/` and `retopo/`, this
directory sits outside the shipping-catalog test pin.

**Every workflow below was RUN TO COMPLETION on a real ComfyUI on
2026-07-16.** Plates: `examples/images/` (copy the PNG/JPGs to the root of
`ComfyUI/input/`; the four ACEScg EXRs are read in place by `OCIORead` —
repoint its `source` and `AtlasRegisterPlate.plate_path` at your copies).

## Requirements matrix

- **All:** the Atlas node pack + `[neural]` extras; plates staged as above.
- **OCIO workflows (4):** ComfyUI-OCIO + `opencolorio`, **opencv-python 4.x**
  (5.x dropped the EXR codec — verified live), and the server launched with
  `OPENCV_IO_ENABLE_OPENEXR=1` **before python starts**. On the authoring
  machine `Windows_Run_GPU.bat` carries the env line *without* `set` (it never
  takes effect) — use `Windows_Run_GPU_EXR_fixed.bat`.
- **VLM workflows (4):** a vision model on lmstudio `:1234`
  (`google/gemma-4-12b-qat` preset); everything fails soft without it.
- **Third-party packs:** ComfyUI-KJNodes (rails), ComfyUI-RMBG (SAM3),
  comfyui-inpaint-nodes + `big-lama.pt`, pysssss (ShowText), rgthree
  (Display Any), Comfyui_VideoCombine_Plus, SaveImageExtended.
- **[moge] / [usd] extras** where MoGe depth / USD exports appear;
  `pyinstantmeshes` for the jungle quad retopo.
- **X-ray:** `ATLAS_EXPERIMENTAL=1` + `ATLAS_LARI_PATH` → your LaRI clone
  (no upstream license — research only).

## What the full-coverage sweep found (notes on what isn't perfect)

1. **`AtlasInput.use_vlm=True` did not force 4 band layers when `layers=0`**
   — the force only fires for `layers>0`; you silently get the single-relief
   path. The workflow now sets `layers=4, sky=True` explicitly. (Doc/UX trap;
   candidate for a follow-up: force regardless.)
1b. **VLM per-band SAM scoping can gut a band** — a PARTIAL segment match
   (mid band: 4.6% coverage on this plate) legitimately keeps the scope,
   leaving the rest of the band's range with NO geometry, which exposes the
   behind-band's `fill_occluded` smear. New `AtlasInput.vlm_scope` widget
   (APPENDED — the mid-list first attempt re-proved the positional
   widgets_values rule live): OFF keeps the VLM's bands/geometry but leaves
   layers band-only. The camera-move workflow ships it OFF; band_mid went
   from 6k verts / 4.6% matte to 96k verts / 66% matte.
2. **A VLM band plan could emit a ZERO-WIDTH far band** (`far=[1.0,1.0]`,
   observed live from gemma on the ghost-town plate): the old monotonic
   check accepted `b3=1.0`, the far card collapsed, the bg band ran to
   infinity and its `fill_occluded` diffusion smear outranked every real
   layer (farthest-highest priorities). **Fixed in
   `assessor.staged_layer_bands`** — boundaries must now be strictly inside
   (0,1) and strictly ordered, else all reset to the fixed slots. Pinned by
   `tests/test_assess_image.py`.
3. **`AtlasPredictHiddenGeometry` without `restrict_mask` substitutes far too
   much** (50.6% of the wreck frame, including visible background — the
   node's own report warns). The workflow wires a SAM3 segment of the
   occluder; 23.8% after.
4. **`AtlasMergeGeometry` doesn't dedupe `projection_ground`** — a 3-way
   merge carries 3 coincident ground planes. Harmless under 📽 (identical
   texels by ray); a cosmetic dedupe would still be nice.
5. **`SaveImageExtended` writes ONE file per queue for an IMAGE batch** in
   this configuration — not a per-frame sequence. The authoritative JPEG
   sequence is extracted from the baked `client_data` (the frames are already
   JPEG bytes); the saver is kept as an in-graph preview of frame delivery.
6. **Stale-node verdicts:** `AtlasLoadImageSolveCamera` — deprecation
   candidate (file-path input predates the tensor solve nodes; can't join
   image chains; kept in the solve lab's "legacy corner" with a note).
   `AtlasDepthAnything` — keep, but it is a *preview* (normalized IMAGE,
   metric data discarded); never wire it toward geometry. Every other node
   earned its place.
7. **`AtlasVLMScaleCues` returned zero references on the alley plate**
   (gemma emitted no door/person cues) — the confirm-gated
   `AtlasApplyScaleReferences` correctly adopted nothing. The reference-object
   node (`AtlasReferenceScaleSolve`) remains the dependable tier-1 path.
