# Retopology demo workflows

Three small feature demos for `AtlasExportReliefMesh`'s export-only
retopology widgets (`retopo_method` + friends — see
`docs/dev/atlas_mesh_retopo.md`). Generated by `_generate.py` (edit that, not
the JSON — widget lists are positional), validated by
`tools/validate_ui_workflow.py`, runnable headless via
`tools/run_ui_workflow.py`. Plate: `atlas_monument_valley.png` at the root of
`ComfyUI/input/`.

Like `examples/experimental/`, this directory is deliberately outside the
shipping-catalog pin (`tests/test_example_workflows.py` globs `examples/*.json`
non-recursively).

**All three run to COMPLETION 2026-07-16** on a real ComfyUI (monument valley
plate, V2-Outdoor depth, grid 256 derive). Measured results:

| Workflow | What it shows | Measured |
|---|---|---|
| `atlas_retopo_quad_dcc_handoff_workflow.json` | hole-fill (48) → **quad** retopo (pure quad, crease 25°) + USD camera | fill: 12 holes +99 faces, 2 loops left (frame is one). Retopo: 36,772 → 12,481 verts / 72,321 → 24,208 faces. OBJ 1:1 v/vt, all UVs in [0,1] |
| `atlas_retopo_decimate_budget_workflow.json` | **decimate** to a 1500-vert budget + dense-mesh viewport | 36,772 → 1,554 verts / 72,222 → 2,999 faces — lands on the ~2× face target exactly |
| `atlas_retopo_smooth_ab_workflow.json` | **smooth** (Taubin ×12) A/B against the OFF baseline | identical topology both sides (9,346 verts / 18,168 faces), UVs untouched |

**Honest note on the quad target:** Instant Meshes treats
`retopo_target_vertex_count` as a *hint*, not a contract — with
`pure_quad=True` + crease constraints the monument-valley mesh came out at
~4× the requested 3000 verts (pure-quad extraction subdivides). Use
`decimate` when you need a hard budget; use `quad` when you need clean quad
flow for sculpting/subdiv and let the count land where the orientation field
puts it.

Deps: `pip install pyinstantmeshes` (quad), `pip install fast-simplification`
(decimate), `pip install trimesh` (smooth). Each path fails with an
actionable ImportError when its dep is absent; the workflows still load.
