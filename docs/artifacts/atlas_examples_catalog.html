Atlas Camera · v0.6.0 beta

Examples Catalog — two workflows, four plates

Atlas ships deliberately lean: one workflow that gets a photo projected onto 3D in a single queue, one that runs the full layered matte-painting pipeline with gates and per-layer debugging — plus a four-image sample pack chosen from the plates that produced the best measured results during development.

The shipped workflows

The catalog below documents the two core entries in examples/. Since the v0.6.0 beta there are more: a third core workflow (atlas_input_ocio_quickstart_workflow.json — the ACEScg float/OCIO handoff: OCIORead → RegisterPlate → AtlasInput → Nuke/Maya/USD exports pointing at the ORIGINAL EXR), plus three run-verified companion sets: examples/showcase/ (11 workflows covering every Atlas node, each with an illustrated deep-dive — see the showcase field guide), examples/retopo/ (export-only quad/decimate/smooth retopology), and examples/experimental/ (X-ray / RenderFix, env-gated). Drag any JSON onto the ComfyUI canvas, point the Load Image node at a sample plate (or any photo), and Queue.

🎬 Quickstart start here

atlas_input_quickstart_workflow.json

Four nodes: LoadImage → AtlasInput → Atlas Viewport. The defaults give instant relief — solve, shared depth, one high-resolution mesh, projected photo in the viewport on the first queue. Every master-workflow behavior is then a widget on the one node:

  • layers 2–4 — watertight depth-band clean-plate layers, with the seam smear baked onto the layers behind (the front band keeps a clean matte)
  • sky — SAM sky card; the mask also scopes every band
  • use_vlm — a local VLM reads the photo and fills prompts, band boundaries and per-band geometry
  • inpaint — per-band LaMa clean plates (crop→inpaint→stitch)

Missing optional packs never error — the feature skips and the report output says so.

🏗️ Staged Master the full pipeline

atlas_camera_staged_master_workflow.json

The same logic opened up into stages: VLM pre-flight → solve gate → SAM sky card → four depth-band stages (each with scoped masks, LaMa clean plates and a cut-out layer preview) → master viewport → Nuke + Maya layer exports, with a 🔍 debug node writing a machine-readable JSON report every run.

Work one layer at a time with the group bypasser, approve the solve once, and assemble when every layer previews clean.

  • Requires: rgthree-comfy, ComfyUI-RMBG, comfyui-inpaint-nodes (+ big-lama.pt), ComfyUI-KJNodes
  • Optional: a local VLM (LM Studio / Ollama) or an OpenAI-compatible key for the pre-flight

The sample pack

Four 4K plates, distributed separately from the repo. Drop them into ComfyUI’s input/ folder with these exact names and both workflows’ defaults line up. Each earned its place with measured results, not vibes.

Which plate for which workflow? The two outdoor plates are the staged master’s sweet spot (real depth bands + a sky to separate); the two interiors show off the quickstart’s single-mesh relief — and prove the skyless path degrades exactly as designed.

The removed examples

Earlier betas shipped ~39 example workflows — merge scenarios, hidden-geometry heroes, OCIO proofs, calibration tests. They were trimmed to the two above for the public release, but every one of them is a single command away:

git show 10e600b:examples/<workflow-name>.json > recovered.json
