The mental model — three sentences

One camera, one photo, many surfaces. Atlas recovers the camera that took the photo, then projects that photo (like a slide projector at the recovered position) onto 3D surfaces. At the recovered view everything reassembles pixel-perfectly; layers only earn their keep when the render camera moves.

Layers stack back-to-front and each carries its own plate. The base mesh gets a (clean-plated) photo; the foreground layer gets the pure original; the X-ray layer gets an inpainted plate; the sky dome gets a clean sky. Z-buffering + priorities decide who paints each pixel.

Masks come in three quality tiers. segmentation (SAM — crisp, semantic) > depth-band (robust, but steppy: DA3 works at ~504px internally) > prediction (the X-ray's own mattes). Use the highest tier available for matte edges; bands are fine for deciding membership.

The staged master — the rig that configures itself

The production version of everything below is atlas_camera_staged_master_workflow.json: five fixed layers (sky + far/bg/mid/fg depth bands), each its own bypasser-switched stage, with a local VLM filling in the per-image decisions on the first queue — segmentation prompts, per-band geometry (relief / card / ground), and band boundaries that stay watertight by construction. Three behaviors distinguish it from the hand-built stacks this guide teaches:

STAGE 1Understand the backdrop before adding anything

The base mesh tears at depth discontinuities — deliberately. Torn quads are the openings the deeper layers will show through. The backdrop (a flat plane at the scene's far depth, emitted automatically with the base mesh) is the designed backstop that fills those openings at camera view.

The tear trade, in one rule

depth_edge_rel low → more tears → more black gaps but honest silhouettes and open reveal paths. High (3–5) → mesh rubber-sheets instead → clean static view, but the stretched sheet can block X-ray reveals behind it. The shipped 1.5 + backdrop ON is the balanced default. Toggling 🎬 Backdrop off without replacing its job = black staircase bands. That is the system working.

The debug lenses — tune with your eyes

LensQuestion it answersWhat good looks like
🎨 LayersWhich layer paints this pixel? Teal world, orange subjects, blue only on reveals, sky pale. Black = nothing paints — always a finding.
🩻 X-rayWhich pixels are invented geometry? Red (LaRI) / blue (WT) only off-angle, never at camera view.
Confidence reportDid the backend register? rel MAD < 0.2 good · 0.2–0.5 shaky · > 0.5 don’t use — switch backends.
🎨 layer cutouts (staged master)What will this layer project? The plate's pixels inside the matte, legend color outside — an all-color cutout means the layer carries nothing.
🔍 debug report (staged master)Anything structurally wrong? FLAGS (0). Zero-vertex layers, band gaps, scope fallbacks and negative depth are named before you go squinting.
🧭 Safe ZoneHow far can the camera actually move? Measured orbit clamps; if the arc is tiny, fix coverage before tuning anything else.

Knob priority — touch them in this order

#KnobNodeWhat it really does
1far_pct (restrict + fg band)Depth Layer Mask / FG layer The scene decision: what counts as “occluder”. Check with the mask preview — subjects fully inside. Everything downstream keys off this. (Staged master: the VLM proposes this — override via band_override.)
2model🔬 Per-scene, not per-taste: LaRI = fast, wins interiors + steep perspective; WT = 20 s diffusion, wins where LaRI’s report says “poor”.
3clear_rel🔬 How far behind the visible a prediction must sit to count. Shallow scenes need low (0.10); deep scenes tolerate 0.30. Symptom of too-high: near-zero substitution.
4SAM promptsSAM3 nodes Free-text — concrete nouns beat expanses (“sand” works where “desert floor and boulder” scores 0.00%). “sky” for the dome; the fg prompt names your occluders.
5relief_grid / depth_edge_relbase + layers Mesh density / tear threshold. Shipped 384 / 1.5 is calibrated — move only with a specific symptom (see below), not for taste.
6smooth_px / fill_gaps🔬 Prediction coherence. Defaults (31 / on) came from measurement; turning fill_gaps off on fragmented scenes (foliage) shreds the X-ray layer.

The one-paragraph tuning loop

Queue at defaults → open 🎨 Layers and fix any black or wrong-color region by adjusting the fg band (and SAM prompts) until subjects are solid orange → read the report; if rel MAD is poor, switch backend → orbit with 🩻 to judge reveals; adjust clear_rel only if the X-ray is starved → bake the dolly. Never tune more than one knob per queue.
