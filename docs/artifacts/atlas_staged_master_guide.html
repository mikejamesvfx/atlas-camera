The Staged Master, Operated

atlas_camera_staged_master_workflow.json splits one photo into five projection layers — sky plus four depth bands — and a local VLM configures it per image: segmentation prompts, per-band geometry, band boundaries. This page is how to drive it: the queue rhythm, what the autopilot fills in, what stays your call, and how to read the debug surfaces when a layer misbehaves.

Queue 1 Assess + solve + preview run in one go. Read the 🧭 report on the assess node, check the camera in the preview viewport, then ✅ Approve Solve.

Queue 2 The enabled stages run. Work far → near with the group bypasser, or enable everything for a full pass. Finished stages re-queue for free (cached).

The autopilot — what the VLM fills in, and what stays yours

On Queue 1 the 🧭 assess node sends the photo to your local VLM with the staged workflow's full decision rules. Three families of values come back and flow into the graph through rails — every receiving widget box updates so you can see exactly what landed:

What flowsWhere it landsThe contract
SAM prompts (5)the sky segmenter + each band's 🎯 scope row Concrete nouns only ("rock formations", never "background"). An absent layer — no sky indoors, an empty mid band — arrives as an empty prompt and its scope self-disarms.
Geometry types (4)each band layer's geometry_override relief / card / ground per band (see below). A value the node doesn't recognize degrades to the widget instead of erroring the graph.
Band boundaries (4)both band nodes per stage via band_override The VLM may move the fixed 30/60/80% splits to keep a subject whole. Boundaries are derived jointly, so adjacent bands always share edges exactly — bands can't drift apart.

What stays yours: approving the solve (the ✅ gate ships closed), which stages are enabled (the bypasser lists stages 1–6 only), and every value above — unlink a rail and type to take a widget manual. The report's STAGED 5-LAYER PLAN marks each layer + (present, with its prompt and geometry) or − (absent).

Gates, current defaults

The assess node ships auto_continue = ON: Queue 1 runs straight through to the solve preview with no ▶ click — the ✅ solve gate is the first stop. Turn it OFF to restore the hard per-image ▶ Continue gate. After a successful assessment the VLM is offloaded from VRAM and the report proves it (VRAM free 1.5 → 12.7 GB on a real run).

Geometry types — relief, card, ground

Band membership always comes from the real depth (which pixels belong to the layer); the geometry type only changes where those pixels sit. Mattes, edge-extend and outpaint work identically on all three.

TypeWhat it buildsUse whenWhy
relief (default)depth-following mesh anything with real 3D shape inside the band Honest parallax within the layer; tears at silhouettes are load-bearing.
cardONE flat plane at the band's median depth distant or flat-facing content — horizon mountains, a hangar's back wall, a skyline Depth noise on far content becomes wobble on what should be rigid; a card can't tear and can't wobble. Rule of thumb: if the layer would move as one poster, card it.
groundthe exact analytic Y=0 plane flat horizontal surfaces the camera stands over — desert floor, water, roads, plazas Every vertex lands on the true ground plane — zero depth-noise bumps by construction.

When LaMa is still not enough

LaMa continues texture. A structured reveal (a building half-hidden behind another) wants the generative tier — swap that one band's LaMa for LanPaint or flux-inpainting and leave the rest of the chain alone.

Self-healing scopes — band ∩ segment, safely

A depth band alone slices through objects; each band's 🎯 scope row intersects the band with a SAM segment of that layer's content. The rows are always active and disarm themselves — the failure modes that used to silently empty a layer to zero mesh now fall back to plain band membership, and the status says which path fired:

scoped to 'distant mountains' (15.3% segment, grown 16px) scoped to 'valley floor and small settlements' (23.9% segment, grown 16px) band-only FALLBACK — segment for 'rolling hills and valley' covered 0.00% of the frame (no-match); scoping skipped so the layer keeps its full band band-only (no scope prompt — layer absent or unscoped)
Prompt craft, from real runs

Text-prompted SAM is good at things and bad at expanses: "church tower" 2.4% ✓, "distant mountains" 15.3% ✓ — but "desert floor and boulder" and "rolling hills and valley" both scored 0.00%. When a wanted scope falls back, retry with a shorter concrete noun ("sand", "hills"). A fallback is never broken output — just band-only behavior.

Reading the debug surfaces

SurfaceWhereWhat it answers
🎨 strip cutoutsleft of the stages, one per layer Exactly what each layer will project: the plate's pixels inside its matte, the layer's legend color everywhere else. A cutout that's all color = the layer carries nothing.
🎯 status lineseach scope row / the 🔍 report Scoped, fallback, or absent — per band, in plain words.

Check flags before judging anything visually — and paste the JSON path at your AI assistant instead of screenshots.

Knob priority — touch them in this order

#KnobWhereWhat it really does
1stage enablesFast Groups Bypasser Work far → near, one stage per queue while learning a shot. Finished stages cache free — never bypass a finished layer (bypass = removed from the stack).
2SAM promptsscope rows (auto-filled) Only when a 🎯 status shows an unwanted fallback — shorten to a concrete noun.
3context_pad_pxeach ✂ crop Per-band fill quality vs context. Drop toward 64 on a band whose plate preview looks soft.
4geometry_override / band_overrideband layers (auto-filled) Override the VLM's call when your eye disagrees — e.g. force relief on a "card" band you intend to dolly toward.
5relief_grid / depth_edge_relband layers Shipped 384 / 1.5 is the band-mesh calibration. Move only on a specific symptom.
6distance_m + radius_msky card Only when authoring a real camera move: distance ≈ 3–5× scene depth, radius sized to the move's widest framing.

Troubleshooting — symptom → cause → fix

You seeIt meansDo
A layer is missing / a region is black That layer built no geometry — check 🔍 flags for ZERO vertices and the 🎯 statuses Absent layer (mid, usually) is fine. Unwanted fallback → fix the prompt. Everything else → read the flag text; it names the cause.
Stripes/holes at band seams when orbiting Band edges drifted apart (🔍 flags: band GAP) Shouldn't happen with the shipped graph (shared band population + joint boundaries). If you rewired excludes: put the plain sky mask on every band node's band_ref_mask.
Inpainted fill looks soft/mushy The 256² budget spread too wide Drop that band's context_pad_px to 64; confirm the upscaler is wired; structured content → generative tier for that band only.
Far layer wobbles on orbit Depth noise on distant content, geometry is relief That's the card case — set the band's geometry to card.
Ground has lumps/bumps Depth noise on a flat floor Geometry ground for that band — analytic plane, zero noise.
Different plates every queue A seed control on randomize Set LaMa/generative seed controls to fixed.
OOM / crawling during stages The VLM still resident alongside SAM/DA3/LaMa Keep offload_model ON and verify the report's VRAM line moved. llama.cpp can't offload — restart its server, or use LM Studio/ollama.
New photo ran with old settings A stale approval Gates are fingerprint-scoped and re-arm on image change by design — if you overrode with a manual proceed, clear it.

The one-paragraph operating loop

Queue → read the 🧭 plan (+/− per layer, prompts, geometry, bands) → ✅ approve the solve → enable stages far-to-near, checking each 🎨 cutout as it lands → any surprise, read 🔍 flags and 🎯 statuses before touching a knob → knobs in priority order, one per queue → when the master viewport holds up under your intended move, export Nuke/Maya layers.
