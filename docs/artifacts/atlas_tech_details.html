Relief mesh mechanics — why 384 / 1.5

The relief mesh tears a quad into a hole whenever its corner depths differ by more than a threshold. That test is load-bearing (silhouettes must not rubber-sheet onto the background), but noisy monocular depth trips it where no real edge exists. The tear test:

tear if  max|zcorner − zcorner| > depth_edge_rel × cell_size(z) Relative to the cell's own world size, so one threshold behaves the same at 3 m and 300 m.
Why band layers ship at 384 / 1.5, not 128 / 0.5

Band-clipped meshes tear at band boundaries on top of silhouette tearing, so the per-layer defaults moved to grid 384 with the threshold relaxed to 1.5 — safe inside a band because the band clip already bounds the mesh's depth range. At the generic 128/0.5 defaults the hangar's ship was mostly holes; three live calibration iterations landed 384/1.5.

Depth backend — the DA3 default flip (since reverted)

Historical note: this flip shipped 2026-07-09 and was reverted on main 2026-07-13 after the broader 4-scene A/B (see the update box above). The measurements stand; the conclusion drawn from them changed when more scenes joined the set.

DA3METRIC emits canonical depth; Atlas converts it to metres using the focal the camera solve already recovered — closing a loop V2's fixed-FOV metric heads structurally can't:

zmetres = focal_px@processed res × canonical / 300 The solved focal comes from GeoCalib or vanishing points — independent of any depth model — so the depth scale inherits the camera solve's accuracy.

X-ray registration — one scale moves the whole stack

A layered-ray model predicts its depth stack in its own arbitrary units. Because all layers share the same rays, registering layer 0 (the visible surface) against the pipeline's DA3 depth aligns every hidden layer with a single robust scalar:

s = median( zpipeline / zlayer0 )  ·  quality = median( |zpipeline − s·zlayer0| / zpipeline )  ← rel MAD rel MAD is the shipped go/no-go: under 0.2 the backend registered; above it, its geometry would corrupt the layer — switch backends.

Selection then walks each ray's layer stack for the first surface that genuinely clears the occluder by a scene-adaptive margin:

hidden(px) = first layer with  z > zvisible + max( clear_rel × zvisible,  min_clear ) Fixed margins fail on shallow scenes — the hangar needed clear_rel 0.10 (coverage 3%→25%); deep scenes tolerate 0.30.

The coherence pass — why gaussian, not median

Raw per-pixel layer selections are fragmented; adjacent pixels picking different layers create ~15 cm depth steps that trip the mesh's world-edge check everywhere — measured immune to depth_edge_rel (identical at 1.5 vs 3.0), grid density, and mask dilation. The fix operates at the source: fill_gaps diffuses the predictions into one surface (dual-field Jacobi on an 8× downsampled grid), then smooth_px smooths the seams.

How to read this page

None of these numbers is a universal constant — they're calibrations on this test set, kept honest by being reproducible: the tear curve re-measures in seconds (build_relief_mesh on synthetic depth), the backend comparison is tools/compare_depth_backends.py, and every 🔬 run prints its own rel MAD. When your scene disagrees with a default, trust your scene's number.
